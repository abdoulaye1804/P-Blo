### рҹ“Ӣ Policy-as-Code вҖ” rГЁgles en YAML

```yaml
# policy-engine/policies/idle-pods.yaml
name: idle-pods
enabled: true

conditions:
  - field: cpu_usage_percent
    operator: lt
    value: 5.0
  - field: age_hours
    operator: gte
    value: 24

actions:
  - type: notify
    config:
      message: "Pod idle : {namespace}/{name} вҖ” CPU: {cpu_usage_percent}%"
      require_approval: false

  - type: scale_down
    dry_run: false           # вҶҗ passer Г  true pour dГ©sactiver
    config:
      replicas: 0
      require_approval: true # вҶҗ validation Slack obligatoire
```

> **Le moteur recharge les politiques Г  chaque cycle** вҖ” modifie un YAML, l'effet est immГ©diat sans redГ©ploiement.

## рҹ“Ӣ Politiques disponibles

| Politique | Ressource | Condition | Action | Dry-run |
|---|---|---|---|:---:|
| `idle-pods` | Pod K8s | CPU < 5% ET RAM < 10% depuis 24h | Notify вҶ’ Scale to zero | вң… |
| `oversized-ec2` | EC2 instance | CPU moy. < 10% sur 24h | Notify вҶ’ Snapshot + Stop | вң… |

### Ajouter ta propre politique en 5 minutes

```yaml
# policy-engine/policies/ma-politique.yaml
name: my-policy
version: "1.0"
enabled: true

scope:
  resource: ec2_instance
  exclude_tags:
    Environment: prod       # вҶҗ jamais d'action en prod

conditions:
  - field: avg_cpu_percent
    operator: lt
    value: 5.0

actions:
  - type: notify
    dry_run: false
    config:
      channel: slack
      message: "Instance trГЁs idle : {instance_id} вҖ” CPU: {avg_cpu_percent}%"
      require_approval: true

schedule:
  interval_minutes: 120
```

> Le Policy Engine dГ©tecte le nouveau fichier au prochain cycle вҖ” **aucun redГ©ploiement nГ©cessaire**.

## рҹ”җ SГ©curitГ©

### 3 niveaux de protection contre les actions accidentelles

```
Niveau 1 : dry_run: true dans le YAML
           в””в”Җв–¶ Log uniquement, aucune modification rГ©elle

Niveau 2 : require_approval: true
           в””в”Җв–¶ Bouton Slack obligatoire avant exГ©cution

Niveau 3 : exclude_tags: Environment: prod
           в””в”Җв–¶ Politique ne s'applique jamais aux ressources de prod
```

### Principe de moindre privilГЁge (IRSA)

| Service Account | Permissions AWS |
|---|---|
| `collector-sa` | Lecture seule : Cost Explorer, CloudWatch, EC2 describe |
| `policy-engine-sa` | Actions EC2/EBS **uniquement** sur ressources taguГ©es `ManagedBy: finops-autopilot` |
| `cluster-autoscaler-sa` | Gestion Auto Scaling Groups uniquement |
